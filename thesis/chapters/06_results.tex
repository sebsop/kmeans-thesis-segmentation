\chapter{Results and Comparative Analysis}\label{chap:results}
{\itshape Comparing the classical and quantum-inspired K-Means engines requires evaluating execution latency, throughput, and GPU utilization against clustering quality metrics (WCSS, the Davies-Bouldin ($\mathcal{D}\mathcal{B}$) index, and Silhouette score). This evaluation includes analyzing visual segmentation quality and temporal stability under varying lighting conditions.}

\section{System Throughput and Latency Scaling}
Execution throughput and stage latency scale as functions of both the target cluster count $K$ and the spatial downsampling stride $S$. These parameters dictate the arithmetic workload and total threads scheduled per iteration.

\subsection{Hyperparameter Latency Benchmarks}
Evaluating GPU scaling under heavy workloads \cite{farivar2008parallel, cook2012cuda} involved testing the algorithms on a static $640 \times 480$ frame of a textured workspace under constant lighting. The scene contains small objects, textures, and color gradients, requiring more iterations to converge than a less textured, uniform video feed. The K-Means++ algorithm \cite{arthur2007k} was used for initialization to ensure identical starting centroids for both engines. Table~\ref{tab:latency_benchmarks} compares the average execution latency and convergence iteration counts over 21 runs.

Execution time scales non-linearly with $K$ and $S$. In the low-density ($K=3, S=8$) and balanced ($K=8, S=4$) configurations, the quantum-inspired engine is slightly faster than the classical engine ($1.61\text{ ms}$ vs. $1.81\text{ ms}$ and $6.15\text{ ms}$ vs. $6.45\text{ ms}$). In the high-density configuration ($K=40, S=1$), which processes all $307,200$ pixels without downsampling, the classical engine is faster ($26.66\text{ ms}$ vs. $27.95\text{ ms}$). Despite the trigonometric math in the quantum-inspired projection, execution latency is similar in all configurations.

\subsection{GPU Latency-Hiding and Hardware Intrinsics}
Three main architectural features of NVIDIA GPUs explain the small latency difference:

\begin{enumerate}
    \item \textbf{Special Function Units (SFUs):} As detailed in Section~\ref{subsect:intrinsics}, the quantum-inspired engine offloads trigonometric calculations to the GPU's dedicated SFUs using the hardware-accelerated intrinsic \texttt{\_\_cosf()} (see Code Snippet~\ref{fig:quantum_sim_code}). This hardware-level execution bypasses standard ALU arithmetic emulation, keeping phase projection overhead low \cite{CudaGuide, cook2012cuda}.
    \item \textbf{Memory Bandwidth Bottleneck:} Global memory access (reading pixel values and centroid locations) dominates the execution time for both engines. Since the GPU hides arithmetic latency by scheduling threads while waiting for memory fetches, the additional trigonometric calculations in the quantum-inspired engine do not significantly slow down overall execution \cite{CudaGuide, farivar2008parallel}.
    \item \textbf{Camera I/O Bottleneck and GPU Utilization:} On an NVIDIA GeForce RTX 5070 Ti, average GPU utilization is approximately 40\% (spiking to 43\%) in the heaviest configuration ($K=40, S=1$). The camera's hardware frame rate (capped at 30 FPS) limits this utilization. The camera delivers a new frame every $33.3\text{ ms}$, and the engine requires only $26.6\text{ ms}$ (classical) or $27.9\text{ ms}$ (quantum) to process it, leaving the GPU idle for the remainder of the frame interval. In the lighter configuration ($K=3, S=8$), the core execution time is under $2\text{ ms}$, leaving the GPU idle for over 90\% of the frame window. Average GPU utilization here is 30-35\%, representing the overhead of rendering the ImGui interface and maintaining the OpenGL window context. This constraint requires using offline throughput calculations to estimate the maximum performance boundaries of the hardware.
\end{enumerate}

\section{Centroid Convergence and Optimization Landscape}
The number of iterations required to reach final numerical stability depends directly on how each engine's distance metric shapes the optimization gradient and drives the centroid update path.

\subsection{Iteration Count and Convergence Velocity}\label{subsect:convergence_velocity}
The number of iterations required to reach stability ($\Delta\boldsymbol{\mu} < 10^{-5}$, representing standard numerical convergence criteria) was measured under the balanced configuration ($K=8$, $S=4$). Using identical initialization seeds and input frames, the classical engine required an average of $41.81$ iterations to converge, while the quantum-inspired engine converged in $41.43$ iterations.

Other configurations show similar behavior: the low-density configuration ($K=3, S=8$) required $11.71$ iterations (classical) and $12.38$ iterations (quantum), while the high-density configuration ($K=40, S=1$) took $88.67$ iterations (classical) and $88.62$ iterations (quantum). Centroid displacement per iteration shows that both engines follow similar descent paths; the quantum-inspired non-linear mapping does not degrade convergence stability or cause oscillations.

\subsection{Hilbert Space Topography}
The convergence profiles described in Section~\ref{subsect:convergence_velocity} reflect the topography of their respective optimization landscapes. In classical Euclidean space, distance scales linearly with feature displacement. In contrast, the quantum-inspired engine projects the normalized 5D features onto a high-dimensional Hilbert hypersphere via the angular phase encoding derived in Section~\ref{subsect:phasespace}. The resulting non-linear distance metric yields a gradient that adapts dynamically based on feature proximity: flat near the centroids ($\theta_d \to 0$), and steep near the cluster boundaries ($\theta_d \to \pi/4$).

While this gradient structure was hypothesized to filter noise and alter boundary behavior (Section~\ref{subsect:boundary_justification}), empirical testing shows it is functionally equivalent to classical Euclidean space. Visual segmentations are identical across both lighting conditions (Section~\ref{subsect:cross_dataset_quality_telemetry}, Section~\ref{subsect:case_study_1}, Figure~\ref{fig:case_study_1}). The WCSS, Davies-Bouldin ($\mathcal{D}\mathcal{B}$) index, and Silhouette scores in Table~\ref{tab:quality_metrics} are also nearly identical.

This equivalence is due to the monotonic nature of the cosine-squared similarity within the bounded phase range $[0, \pi/4]$. Since the mapping preserves the relative ordering of feature distances, both engines converge to the same local minima. Thus, the quantum-inspired topography is a mathematically valid alternative formulation that can be simulated in real-time with negligible overhead, maintaining the stability, accuracy, and output quality of classical K-Means.

\section{Quantitative Quality Benchmarking}
Comparing the mathematical accuracy of the classical and quantum-inspired engines requires evaluating their outputs under identical lighting conditions using quantitative clustering metrics.

\subsection{Definition of Evaluated Quality Indices}
The segmentation quality of both engines is evaluated using three standard diagnostics (Section~\ref{subsect:metrics_def}):
\begin{enumerate}
    \item \textbf{Within-Cluster Sum of Squares (WCSS):} Measures the compactness or quantization error. A lower WCSS indicates that pixels are closer to their centroids.
    \item \textbf{Davies-Bouldin ($\mathcal{D}\mathcal{B}$) Index:} Evaluates the ratio of cluster dispersion to centroid separation. A lower index indicates better-separated clusters.
    \item \textbf{Silhouette Coefficient:} Gauges individual pixel assignment fit. A Monte Carlo approximation with $M=2,000$ random samples is used to avoid the high computational cost of the exact score (Section~\ref{subsect:silhouette_complexity}). A higher score represents better-defined clusters.
\end{enumerate}

\subsection{Cross-Dataset Quality Telemetry}\label{subsect:cross_dataset_quality_telemetry}
The quantitative benchmarks were evaluated on a static VGA ($640 \times 480$ pixels) video frame captured under two environments (Figure~\ref{fig:scenes}):
\begin{itemize}

    \item \textbf{Scene B (Low-Light):} A dimly lit room with monitor glow, creating sensor noise and color gradients.
\end{itemize}

\subsection{Non-Linear Class Discrimination}
Table~\ref{tab:quality_metrics} shows that the quantum-inspired engine achieves mathematical parity with the classical Euclidean engine. In Scene A, WCSS values differ by less than $0.003\%$ ($2189.43$ vs. $2189.49$), while the Davies-Bouldin $\mathcal{D}\mathcal{B}$ index ($0.68200$ vs. $0.68194$) and Silhouette scores ($0.48335$ vs. $0.48338$) are nearly identical. Scene B shows similar parity.

This parity indicates that the angular phase encoding is mathematically consistent. Although the quantum-inspired metric maps features onto a non-linear Hilbert hypersphere (which introduces steep boundary gradients and flat gradients near centroids), the class discrimination performance is unaffected. The monotonic property of the squared-cosine similarity preserves the relative ordering of pixel distances during optimization, preventing color-gradient anomalies and noise-induced clustering degradation. The non-linear topography is a stable functional equivalent to classical Euclidean distance under both lighting conditions.

Conversely, setting it too high forces the clusters into rigid, grid-like shapes that ignore physical object contours. At the balanced baseline of $10^{-6}$ (Table~\ref{tab:hyperparameters}), both backends suppress noise and preserve edge details, maintaining visual equivalence.

\subsection{Concluding Engineering Verdict}
Since the quantum-inspired engine achieves functional equivalence without latency overhead, it serves as a mathematical baseline for future hybrid quantum integration. Simulating the phase-space similarity on a GPU ensures predictable performance when porting the solver to physical NISQ co-processors.

For edge deployment, the frame-by-frame centroid shifts observed in both engines (Figure~\ref{fig:jitter_sequence}) suggest that temporal consistency can be further managed in future work. Tracking centroids across consecutive frames via temporal association or optical flow tracking represents a potential way to stabilize boundaries and minimize visual label switching.
